Interface Description

The website interface studied in this project is the current students page of Yale university's art school. The screenshot below shows what this webpage looks like. Follow this link to experience this interface: www.art.yale.edu/about/people/current-students

Homepage of Yale School of Art

Usability Analysis

There are a number of obvious usability issues with this website. The usability issues highlighted below were identified through the lens of efficiency, learnability, and memorability as well as general comments.

  • There is no immediately obvious structure and hierarchy to the content of this webpage
  • There is no discernable difference between links and regular text
  • The chosen background image presentation format competes with the actual interactive elements and informative text of the website for attention.
  • Listing the names of every single student makes the webpage very text-heavy and overwhelming to parse.
  • There is no quick and efficient method to find the name of a student.
  • Very relevant information sections and links are hidden below the complete list of student names and require users to scroll a lot to see them.

Low-Fidelity Prototypes

Below are the low fidelity prototypes designed to address the various usability issues identified in the webpage under consideration. The wireframes were designed for three screen sizes (Desktop, Tablet and Mobile). The wireframe for the Desktop size is annotated with comments describing how design choices address the various usability issues. The tablet and mobile interfaces have the same elements as desktop, however they are simply rearranged to responsively cater for the difference in screen sizes.
